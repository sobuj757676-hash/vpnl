1. Executive Summary
This document defines the complete, refined product requirements for a Multi-Tenant VPN Portfolio Ecosystem — a scalable web platform enabling a single organization to manage and showcase unlimited VPN products through a centralized Super Admin Panel, a world-class branded main website, and dynamically generated, conversion-optimized subdomain landing pages.
The architecture leverages Wildcard DNS (*.abcd.com) to allow instantaneous creation of new VPN product storefronts without any server-level configuration changes. All content — branding, copy, legal documents, app-store links, and pricing — is managed exclusively through the Super Admin Panel and served dynamically from a shared database and codebase.
Core Value Proposition:  One Super Admin Panel → unlimited individually branded VPN storefronts, zero DevOps friction, 10/10 UI/UX on every page.
Key refinements in this version (v2.0.0):
Role system simplified to Super Admin only — no Editor or Viewer roles.
UI/UX elevated to a 10/10 standard: world-class design system for main site and all subdomain landing pages.
Admin panel streamlined with a clean, modern interface exclusively for the Super Admin.
All API endpoints consolidated under Super Admin authentication.

2.2  High-Level Architecture
Layer
Description
DNS / CDN Layer
Wildcard DNS record (*.abcd.com) routes all subdomain traffic to the same server. Cloudflare CDN caches static assets per subdomain.
Reverse Proxy / Router
Reads the Host header from the incoming request and extracts the subdomain slug (e.g., 'laion' from laion.abcd.com).
Application Server (Next.js)
Uses the slug to query the database for the corresponding VPN product record and server-renders the landing page template with fetched data.
Database (PostgreSQL)
Single source of truth for all VPN product data, branding tokens, legal content, analytics events, and Super Admin configuration.
Super Admin Panel (React)
Communicates with a REST API secured by JWT. All CRUD operations reflected immediately on the live site.

4.6  Performance as a Design Requirement
LCP (Largest Contentful Paint): < 1.5s — hero image must be preloaded with priority attribute.
CLS (Cumulative Layout Shift): < 0.05 — all images must have explicit width/height; skeleton placeholders used.
INP (Interaction to Next Paint): < 200ms — no heavy JS blocking the main thread.
Font loading: font-display: swap with a fallback font stack matching Inter's metrics to prevent FOUT.
Images: all uploads auto-converted to WebP + AVIF with <picture> srcset. Hero images lazy-blurred (blurhash placeholder).
4.7  Accessibility (WCAG 2.1 Level AA — Non-Negotiable)
All interactive elements reachable and operable via keyboard alone.
Focus rings: visible, high-contrast (2px solid, offset 2px) — never removed with outline: none without replacement.
aria-label on all icon-only buttons; alt text on all images (editable from Admin Panel).
Color is never the only means of conveying information.
Skip-to-main-content link at page top (visually hidden until focused).
All form inputs in admin have associated <label> elements.

6. Component 2 — Subdomain Landing Pages ([vpn-name].abcd.com)
Each subdomain is a dynamically rendered, conversion-focused single-product landing page. The same codebase is reused across all products — only the data from the database changes per slug. The design must match the UI/UX quality of leading VPN brands (NordVPN, ExpressVPN, Surfshark). Every element is purposeful.
6.1  Slug Resolution Logic
Request flow for subdomain resolution:
Browser requests: laion.abcd.com
Wildcard DNS routes the request to the application server.
Server reads the Host header and extracts slug: 'laion'.
Server queries: SELECT * FROM vpn_products WHERE subdomain_slug = 'laion' AND status = 'published'.
If found → server-renders the landing page with product data. If not found → renders a branded 404 page.

7. Component 3 — Super Admin Panel (admin.abcd.com)
The Admin Panel is the exclusive control room for the entire ecosystem. Only one role exists: Super Admin. Super Admin has full, unrestricted access to all features. The UI is clean, modern, and professional — built to be fast and intuitive for daily operational use.
7.1  Authentication
Feature
Requirement
Login
Email + password. Passwords hashed with bcrypt (cost factor 12+).
2FA (TOTP)
Mandatory for all Super Admin accounts. Google Authenticator / Authy compatible.
Session Management
JWT-based; access token (15 min TTL) + refresh token (7 days) stored in httpOnly, Secure, SameSite=Strict cookies.
Rate Limiting
Max 5 failed login attempts per 15 minutes per IP. Account temporarily locked after 10 failures in 1 hour.
Session Timeout
Auto-logout after 60 minutes of inactivity. Configurable by Super Admin (30–120 min).
Audit Log
Every create/update/delete action logged with: timestamp, actor email, entity type, entity ID, changed fields (before/after).
New Device Alert
Email notification sent to Super Admin on login from unrecognized IP/device fingerprint.
Session Revocation
Super Admin can view and terminate all active sessions from the Security Settings page.

13. Risks & Mitigations
Risk
Impact
Mitigation Strategy
Wildcard SSL renewal failure — all subdomains go HTTPS-broken
Critical
Auto-renewal with monitoring alerts 30 days before expiry. Test in staging first.
Subdomain slug collision (duplicate slug attempt)
High
UNIQUE constraint at DB level + real-time slug availability check via debounced API call in admin form.
Poor SEO for dynamically rendered subdomain pages
Medium
Next.js SSR/ISR — full HTML with meta tags server-rendered. Per-subdomain sitemaps submitted to Google Search Console.
Admin panel performance degradation with 100+ VPN products
Medium
Server-side pagination, lazy loading, indexed DB queries on subdomain_slug and status columns.
Uploaded images slowing page load
Medium
Auto-convert to WebP/AVIF, generate responsive srcset sizes, serve via CDN with correct cache headers.
GDPR / privacy compliance gap in analytics
High
Hash IP addresses (SHA-256); no PII collected; cookie consent banner; Data Processing Agreement in legal docs.
Super Admin account compromise
Critical
Mandatory 2FA (TOTP); full audit log; session revocation endpoint; email alert on new device login.
Animation causing performance regression
Medium
Intersection Observer for scroll-triggered animations; will-change hints; GPU-composited transforms only; reduce-motion respected.
FOUC (Flash of Unstyled Content) on theming
Medium
CSS custom properties injected server-side during SSR. Font preloaded with font-display: swap.

15. Glossary
Term
Definition
Wildcard DNS
A DNS record matching any hostname within a domain (*.abcd.com matches anything.abcd.com).
Subdomain Slug
The unique identifier in a subdomain URL (e.g., 'laion' in laion.abcd.com). URL-safe, lowercase.
Super Admin
The sole admin role in this system. Has full unrestricted access to all features.
Multi-tenant
One application instance serving multiple clients (tenants) with isolated data and configuration.
SSR (Server-Side Rendering)
HTML generated on the server per request, improving SEO and initial load time.
ISR (Incremental Static Regeneration)
Next.js feature: static pages regenerated in background at a defined interval, no redeploy.
DNS-01 Challenge
ACME protocol method used by Let's Encrypt to verify domain ownership via DNS TXT record — required for wildcard certificates.
FOUC (Flash of Unstyled Content)
Brief period during page load where content appears without its intended styles.
CTA (Call to Action)
A UI element (usually a button) prompting the user to take a specific action.
JWT (JSON Web Token)
Compact, URL-safe token format for securely transmitting authentication claims between parties.
WCAG
Web Content Accessibility Guidelines — international standards for web accessibility.
LCP (Largest Contentful Paint)
Core Web Vital: time for the largest visible content element to load. Target < 1.5s.
CLS (Cumulative Layout Shift)
Core Web Vital: measure of visual stability. Target < 0.05.
Glassmorphism
UI design style using frosted glass effect: translucent backgrounds with blur, soft borders, shadow.
Blurhash
Compact representation of a placeholder blurred image shown while the real image loads.
TOTP (Time-based One-Time Password)
2FA method generating a rotating 6-digit code every 30 seconds (Google Authenticator).
